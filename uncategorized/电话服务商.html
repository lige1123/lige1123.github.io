电话服务商

（线性dp） 动态规划

  1. 定义dp 状态，dp[n][i][j][k]

    代表 n 个请求，最终这三名服务位于第 i, j, k 位置；

    但是显然内存超出；我们考虑降维：

    1. 显然最后一定会有一名服务员位于第n 个请求位置 p[n],所以降低成

      dp[n][i][j] 表示 n 个人，其他两个人在 i位置和 j 位置；

    但是这里的内存还是没有达到要求，我们在递推公式里面再推导；

  1. 递推关系式：

    想要转移到dp[n][i][j] 位置 ，分别是：

    从 i 位置转移到 n 位 ：

    (1) dp[n][i][j] = dp[n - 1][n - 1][j] + c[i][n]

    从 j 位置转移到 n 位置 ：

    (2) dp[n][i][j] = dp[n - 1][i][n - 1] + c[j][n]

    从 n - 1 位置转移到 n 位置：
    (3) dp[n][i][j] = dp[n - 1][i][j] + c[n - 1][n]

    我们的递推关系式则是取上面三个式子的最小值；

    dp[n][i][j] = min((1), (2), (3))

    也正是在这里可以看出，递推关系式 只有n - 1 和 n 两个状态；所以可以声明一个二维的数组；

    把上面式子中的所有 n - 1 n &1 运算;

  2. 边界：

    dp[0][1][2] = 0, 因为数据要相互比较求最小值，故其他的 dp 值应该取程序极大值；

  3. 答案：

    dp[n & 1] 中的最小值；
